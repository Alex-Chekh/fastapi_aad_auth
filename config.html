Configuration Options¶

fastapi_aad_auth configuration options.

class fastapi_aad_auth.config.AADConfig(_env_file: Optional[Union[pathlib.Path, str]] = '<object object>', _env_file_encoding: Optional[str] = None, *, client_id: pydantic.types.SecretStr, tenant_id: pydantic.types.SecretStr, client_secret: pydantic.types.SecretStr = None, scopes: List[str] = ['Read'], client_app_ids: List[str] = None, strict: bool = True, api_audience: str = None, redirect_uri: pydantic.networks.HttpUrl = None, prompt: str = None, domain_hint: str = None, roles: List[str] = None)[source]¶

Configuration for the AAD application.

Includes expected claims, application registration, etc.

Can also provide additional client application ids to accept.

A list of roles can be provided to accept (requires configuring the roles in the AAD application registration manifest)

Keyword Arguments

  • client_id (pydantic.types.SecretStr) -- Application Registration Client ID (Can be set by AAD_CLIENT_ID environment variable)

  • tenant_id (pydantic.types.SecretStr) -- Application Registration Tenant ID (Can be set by AAD_TENANT_ID environment variable)

  • client_secret (pydantic.types.SecretStr) -- Application Registration Client Secret (if required) (Can be set by AAD_CLIENT_SECRET environment variable)

  • scopes (list) -- Additional scopes requested [default: ['Read']]

  • client_app_ids (list) -- Additional Client App IDs to accept tokens from (when running as a backend service) (Can be set by AAD_CLIENT_APP_IDS environment variable)

  • strict (bool) -- Check that all claims are provided [default: True] (Can be set by AAD_STRICT_CLAIM_CHECK environment variable)

  • api_audience (str) -- Corresponds to the Application ID URI - used for token validation, defaults to api://{client_id} (Can be set by AAD_API_AUDIENCE environment variable)

  • redirect_uri (pydantic.networks.HttpUrl) -- The redirect URI to use - overwrites the default path handling etc (Can be set by AAD_REDIRECT_URI environment variable)

  • prompt (str) -- AAD prompt to request (Can be set by AAD_PROMPT environment variable)

  • domain_hint (str) -- AAD domain hint (Can be set by AAD_DOMAIN_HINT environment variable)

  • roles (list) -- AAD roles required in claims (Can be set by AAD_ROLES environment variable)

class fastapi_aad_auth.config.RoutingConfig(_env_file: Optional[Union[pathlib.Path, str]] = '<object object>', _env_file_encoding: Optional[str] = None, *, login_path: str = '/login/oauth', login_redirect_path: str = '/login/oauth/redirect', logout_path: str = '/logout', landing_path: str = '/login', home_path: str = '/', post_logout_path: str = None)[source]¶

Configuration for authentication related routing.

Includes logout_path, login_path, and login_redirect_path (defaults should be fine for most use-cases).

There are also the landing_path for the login page, as well as the home_path for the home page (defaults to the application root), and the post_logout_path for any specific routing once a logout has completed.

Keyword Arguments

  • login_path (str) -- Path for initiating the AAD oauth call [default: /login/oauth] (Can be set by FASTAPI_AUTH_LOGIN_ROUTE environment variable)

  • login_redirect_path (str) -- Path for handling the AAD redirect call [default: /login/oauth/redirect] (Can be set by FASTAPI_AUTH_LOGIN_REDIRECT_ROUTE environment variable)

  • logout_path (str) -- Path for processing a logout request [default: /logout] (Can be set by FASTAPI_AUTH_LOGOUT_ROUTE environment variable)

  • landing_path (str) -- Path for the login UI page [default: /login] (Can be set by FASTAPI_AUTH_LOGIN_UI_ROUTE environment variable)

  • home_path (str) -- Path for the application home page (default redirect if none provided) [default: /] (Can be set by APP_HOME_ROUTE environment variable)

  • post_logout_path (str) -- Path for the redirect post logout - defaults to the landing path if not provided (Can be set by FASTAPI_AUTH_POST_LOGOUT_ROUTE environment variable)

class fastapi_aad_auth.config.SessionConfig(_env_file: Optional[Union[pathlib.Path, str]] = '<object object>', _env_file_encoding: Optional[str] = None, *, secret_key: pydantic.types.SecretStr = '0c5de382-20c7-484e-a410-0b7374ceb63c', session_cookie: str = 'session', same_site: str = 'lax', https_only: bool = False, max_age: int = 86400)[source]¶

Configuration for session middleware.

Contains the session secret (should be set as environment variables in a multi-worker/multi-processing environment to enable authentication across workers)

Provides configuration for the fastapi session middleware

Keyword Arguments

  • secret_key (pydantic.types.SecretStr) -- Secret used for the session middleware [default: 0c5de382-20c7-484e-a410-0b7374ceb63c] (Can be set by SESSION_SECRET environment variable)

  • session_cookie (str) -- Cookie name for the session information [default: session] (Can be set by SESSION_COOKIE environment variable)

  • same_site (str) -- Cookie validation mode for the session [default: lax] (Can be set by SESSION_SAME_SITE environment variable)

  • https_only (bool) -- Allow the sessions only with https connections (Can be set by SESSION_HTTPS_ONLY environment variable)

  • max_age (int) -- Maximum age for a session [default: 86400] (Can be set by SESSION_MAX_AGE environment variable)
